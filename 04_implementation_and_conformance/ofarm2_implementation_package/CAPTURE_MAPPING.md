# Capture-to-contract mapping (spray record)

Status: binding implementation artifact for the capture app (hostile-review finding F4). The farmer touches **five things**; everything else is auto-populated. If a field below ever appears on the farmer's screen as a required text input, that is a design regression.

## What the farmer touches

| UI element | Contract destination | Source / behavior | Failure mode |
|---|---|---|---|
| **Product** (picker, cached register) | `ExecutionRecordPayload.actualAction` product ref + `AgronomicIdentityBinding` (scheme = SI register, value = authorisation id, `referenceSnapshotRefs` = cached snapshot) | Autocomplete over the offline FITO-INFO snapshot; picking creates/reuses the binding + machine-generated `ExternalRegistryVerificationTrace` | Product not in snapshot → binding recorded `UNRESOLVED` + advisory flag; record stays committable as claim, promotion requires review |
| **Dose** (number + unit) | `actualQuantityParameters` (value + UCUM `unitCode` + QUDT quantity kind) | Unit defaulted from product label data where available | Implausible dose vs. label range → advisory flag + routes to advisor review, never silent block |
| **Parcel** (picker) | `executionExtent` scope ref → Field identity; `PartialExtent` if "part of parcel" toggled (area + geometry basis class) | Farm's GERK-bound field list, cached offline | Unknown parcel → draft only; structure assertion needed first (one-tap "add field" flow) |
| **Crop** (confirm) | crop binding ref on the payload (EPPO via profile) | Defaulted from the parcel's auto-created `CropCycle`; one tap to change | No cycle → auto-create (`autoCreated: true`) |
| **Time** (confirm) | `effectiveTimeInterval` (event time) | Defaulted to now; editable for delayed entry — **event time, not record time** | — |
| *(optional)* Photo | `EvidenceRecord` (PHOTO, sha256 digest) linked via `evidenceRefs` | One tap; stored locally, synced later | Missing photo never blocks a claim (profile floor does not require it) |

## What the app auto-populates (farmer never sees)

| Contract field(s) | Auto-fill source |
|---|---|
| `ExecutionRecordPayload`: `recordClass`=OPERATION_CLAIM-equivalent, `recordState`, `capturedAt`, `subject`, `anchorScopes`, `actor` (party + delegation ref if worker), `equipment` (farm's sprayer, last-used default, with register sticker number where matched), `sourcePayload` (the raw capture JSON, verbatim), `promotionBoundary` (non-promotion declaration), `codeBindingProfileRef` | App state + session identity + profile constants |
| SI legal operator fields (ZFfS-1 Art. 44.b(3)): professional user name + address; KMG-MID (or tax no.); **FFS training-card number** (`Party.registeredIdentifiers`, scheme `SI:FFS-IZKAZNICA`) | Captured once at onboarding; auto-attached to every record — never typed per record (see `profile_si_ffs/SI_RECORD_FIELDS.md`) |
| EU Annex conditionals: start hour (when product/use makes time relevant, fn 4); BBCH stage (where relevant, fn 7); fraction of unit treated (fn 2 → `PartialExtent`) | Surfaced conditionally from snapshot data / "part of parcel" toggle; defaulted otherwise |
| `CommitIngressRequest`: `ingressChannel`, `commitClass`="operation claim", `idempotencyKey`, `originatingOfflineQueueRef`, `sourcePayloadDigest` | Sync engine (idempotency key = device + queue sequence; survives retries) |
| `SemanticEventEnvelope`: family=INTERVENTION, `dominantSemanticConsequence`, `timeSemantics` (event time = field entry; record time = server commit) | Commit pipeline |
| `EvidenceSufficiencyCase` (claims/arguments/bundles vs. the SI floor) | Auto-generated from the policy template at promotion — never hand-authored |
| `ExternalRegistryVerificationTrace` (19 required fields: lookup surface, query inputs, candidate count, selection rationale, snapshot refs, availability, outcome…) | Generated by the binding service against the cached snapshot at pick time |
| `ReviewDecision` (self-review) | Created by the deliberate "confirm & accept" step; decider = farmer party |
| `PromotionTrace` + reachability links | Commit pipeline, same transaction |

## Sync rules (offline reality)

1. Drafts queue locally with the **snapshot id they were captured against**.
2. At sync, ingress re-runs authority (revocation re-check — a worker whose delegation was revoked while offline gets review, not acceptance) and re-verifies bindings against the current snapshot (changed authorisation → discrepancy recorded → review).
3. Event time is the field-entry time; record time is the server commit time; the two never collapse (temporal-conformance sub-gate).
4. A failed or conflicted sync leaves a draft + a `RuntimeProblem` the farmer can see — never a half-committed record.
