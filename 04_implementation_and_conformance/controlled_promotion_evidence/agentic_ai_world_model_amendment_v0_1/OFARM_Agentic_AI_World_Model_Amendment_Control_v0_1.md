# OFARM Agentic AI and World-Model Amendment Control v0.1

Date: 2026-05-14  
Status: draft supporting amendment start; not active law until promoted through OFARM authority process  
Role: phase-control and authority-discipline artifact for the Agentic AI / World-Model Readiness Amendment.

## 1. Problem being closed

The active OFARM baseline is conceptually strong for AI assistance, but not yet native to serious multi-agent and world-model runtimes.

The missing pre-implementation layer is not a new truth model. It is an explicit, governed envelope for:

- software-agent identity and sponsorship
- agent authority and revocation
- agent run scope, inputs, tools, and write surfaces
- agent run traces
- agent handoff
- agent capability and tool declarations
- Advisory-only world-model state
- evidence and observation requests
- hostile conformance tests

## 2. Scope of this amendment start package

This start package covers:

- Phase 0: amendment control and authority hygiene
- Phase 1: no-regrets baseline clarification proposals
- Phase 2: promotion map for existing AI/app support material
- Phase 3: draft Agent Actorship and Authority RFC
- Phase 4: draft Agent Run Envelope, Trace, and Handoff RFC
- Phase 5: draft capability-manifest agentic extension
- Phase 6: draft Advisory-only world-model runtime RFC
- Phase 7: draft EvidenceNeed and ObservationRequest RFC
- Phase 8: farmer-value scenario fixtures
- Phase 9: break-test suite
- Phase 10: implementer directions

## 3. Non-goals

This amendment does not:

- rewrite the Constitution
- rewrite the Platform baseline
- make supporting material active law without controlled promotion
- make AI outputs governance decisions
- create a third AI Twin
- make world-model state Compliance current state
- make agent memory canonical truth
- let packs mutate core meaning
- let tool-call success waive semantic-law blockers
- let handoff transfer authority
- claim runtime readiness before implementation evidence exists
