# OFARM Phase 3 Agent Authority Defaults v0.1

Date: 2026-05-14  
Status: supporting review note; not active law

## Default rule

Agent capability must be narrower than or equal to the authority granted to the accountable party or delegator. Agent identity, model capability, operational convenience, or tool-call success never expands authority.

## Human-governed default list

The following action classes remain human-governed by default:

- `REVIEW_ACCEPT`
- `REVIEW_REJECT_OR_CONTEST`
- `REVIEW_SUPERSEDE`
- `CONTEXT_INSTALL_PACK`
- `CONTEXT_ACTIVATE_PACK`
- `CONTEXT_DEACTIVATE_PACK`
- `OUTPUT_APPROVE_DOCUMENT_ASSEMBLY`
- `OUTPUT_ATTEST_DOCUMENT_ASSEMBLY`
- `OUTPUT_FILE_SUBMISSION_ASSEMBLY`

## Agent output categories

Agent work products must resolve into existing OFARM-governed categories. `Generated by agent` is provenance, not a truth family.

## Revocation default

If a profile, instance, sponsor relation, grant, delegation, sharing permission, model/tool approval, or tool permission is revoked or suspended before final action, final action must be denied or re-routed through explicit review/human approval.

## Permission-limited answers

An agent may not treat a permission-limited answer as proof that no record exists. The result must remain qualified and traceable.
