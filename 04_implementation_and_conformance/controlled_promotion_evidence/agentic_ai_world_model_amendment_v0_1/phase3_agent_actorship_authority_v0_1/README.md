# OFARM Agentic AI and World-Model Amendment — Phase 3

Date: 2026-05-14  
Status: **supporting pre-implementation amendment package; not active baseline law**  
Phase: **Agent Actorship and Authority**

This phase hardens the existing draft `OFARM_Agent_Actorship_and_Authority_RFC_v0_1` into a fuller review candidate and adds machine-contract drafts, examples, negative cases, and conformance directions for governed software-agent identity.

## What this phase does

- Defines a reviewable distinction between accountable party, software-agent profile, deployed agent instance, sponsor, model/tool profile, authority envelope, and revocation state.
- Adds agent-specific authorization decision trace requirements.
- Maps all active `AuthorityActionClass` values to recommended default agent posture.
- Provides draft machine contracts and examples for validation.
- Preserves human-governed defaults for review, pack/context governance, output approval, attestation, filing, and high-consequence compliance promotion.

## What this phase does not do

- It does not promote anything into `00_active_baseline/`, `01_companion_artifacts/`, `02_accepted_rfcs/`, or `03_machine_contracts/`.
- It does not create runtime readiness, production readiness, or two-agent compatibility claims.
- It does not authorize autonomous compliance decisioning.
- It does not create an `AgentOutput` truth bucket.

## Exit posture

This phase is ready for architect review as a candidate accepted-RFC package. It should be promoted only after review against the active Authority Policy Model, Authority Action Matrix, Platform enforcement architecture, and any future Phase 4 run-envelope/handoff work.
