# OFARM Phase 4 Run Governance Principles v0.1

Status: supporting review note; not promoted

1. A run is a bounded governed context, not authority.
2. A run envelope constrains what may be attempted.
3. A run trace explains what happened.
4. Tool success is not governance success.
5. Output disposition resolves to existing OFARM categories.
6. Handoff transfers context, not authority.
7. Stale, redacted, partial, advisory, and permission-limited input remains qualified.
8. Revocation, failed preflight, and missing approval block or degrade final writes.
9. Agent scratch memory is not canonical truth.
10. Runtime conformance cannot be claimed until implementation evidence exists.
