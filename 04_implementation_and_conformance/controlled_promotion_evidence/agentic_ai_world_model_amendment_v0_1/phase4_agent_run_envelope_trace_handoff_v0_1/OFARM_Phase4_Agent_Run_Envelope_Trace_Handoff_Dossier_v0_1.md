# OFARM Phase 4 Agent Run Envelope, Trace, and Handoff Dossier v0.1

Date: 2026-05-14  
Status: supporting review dossier; not promoted

## Purpose

Phase 4 answers how a bounded agent run is started, constrained, traced, blocked, handed off, and disposed. Phase 3 defines who the agent is and how its action is authorized; Phase 4 defines how a multi-step run remains inspectable and non-authoritative unless normal OFARM gates allow a later outcome.

## Active-law fit

The phase preserves assertion/history-first truth, governed materialization, two twins only, QuerySpecification discipline, explicit authority, explicit evidence, review/promotion gates, human-governed defaults, and data-sovereignty boundaries.

## Main controlled patch

The future active patch should accept the run-envelope/trace/handoff RFC, promote the run/handoff schemas, bind tools to preflight and trace surfaces, and explicitly state that handoff transfers context only, not authority.

## New risk closed

The phase closes the risk that multi-step agent sessions become hidden governance channels, hidden current-state stores, or untraceable chains of tool calls.

## Remaining risk

Capability Manifest `agentSupport`, AgentToolManifest harmonization, Advisory-only world-model contracts, EvidenceNeed/ObservationRequest, and runtime conformance remain later phases.
