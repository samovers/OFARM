# OFARM Phase 9 Attached Research Analysis v0.1

The attached research reinforces that Phase 9 should focus on hostile conformance rather than new baseline law. It recommends break tests for spoofed workload identity, sponsor revocation, handoff replay, MCP/tool-hint escalation, external-data auto-promotion, hidden truth-store creation, pack/profile semantic mutation, query widening, sharing bypass, world-model invalidation failure, memory-to-truth leakage, benchmark deanonymisation, missing blocked-action traces, offline contractor late-sync conflict, and BridgeCandidate auto-promotion.

Phase 9 adopts those recommendations as test families. The tests are framed as pre-implementation fixtures, not executed evidence. A future implementer must turn them into executable conformance tests before claiming runtime readiness.

The strongest research-derived addition is that a platform must test not only successful behavior but also refusals and blocked attempts. Missing blocked-action traces are themselves a readiness failure.
