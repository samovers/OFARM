# OFARM Agentic AI Conformance Claim Limits v1.0

## Allowed claims

An implementer or architect may say:

- A pre-implementation supporting amendment package exists.
- Phase AAI-P1 baseline safety clarification is included in the cumulative working copy.
- Candidate RFCs, machine contracts, examples, farmer-value scenarios, and hostile break tests are prepared for review.
- The package is implementation-directed.

## Prohibited claims before implementation evidence

Do not claim:

- runtime readiness;
- production readiness;
- two-agent compatibility;
- autonomous compliance decisioning;
- world-model compliance automation;
- external standard readiness;
- live external registry integration;
- audit certification;
- AI-certified audit readiness.

## Required evidence for stronger claims

A stronger claim requires:

- promoted active contracts;
- implemented public operation surfaces;
- executed preflight/dry-run/explain tests;
- trace retrieval tests;
- sharing and revocation tests;
- world-model invalidation tests;
- offline late-sync tests;
- two independent agent builds;
- all Phase 9 hostile tests passing with expected blocked-action traces.
