# OFARM Phase 6 World Model Governance Principles v0.1

1. A world model is an Advisory Twin runtime capability, not a third twin.
2. `WorldModelState` is not canonical truth, not Compliance Twin state, and not current-state materialization.
3. A scenario result may inform action, but it cannot certify action.
4. Model confidence is not evidence sufficiency.
5. Calibration evidence is not authority.
6. Observed-outcome reconciliation improves model credibility evidence; it does not silently rewrite canonical history.
7. Invalidation is part of the artefact contract, not an optional runtime concern.
8. World-model memory, vector stores, summaries, and cached state remain non-authoritative unless surfaced as governed OFARM artifacts and promoted through normal gates.
9. Pack/profile context may constrain a scenario, but no pack or prompt may use world-model state to mutate core meaning.
10. Every high-consequence use of a world-model output must be blocked unless it passes normal OFARM evidence, authority, freshness, review, publication, and sharing checks.
