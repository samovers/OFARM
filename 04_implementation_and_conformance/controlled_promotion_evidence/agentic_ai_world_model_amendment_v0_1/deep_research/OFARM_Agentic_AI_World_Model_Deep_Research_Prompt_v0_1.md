# Supplemental Deep Research Prompt — OFARM Agentic AI and World-Model Readiness

Research goal: identify current best practices for making a governed crop-farming semantic/runtime platform ready for multi-agent AI and world-model AI without weakening OFARM truth, authority, evidence, traceability, current-state, pack, query, and twin-separation law.

OFARM context:

- assertion/history-first canonical truth
- governed current-state materialization
- one semantic substrate
- Compliance Twin and Advisory Twin
- explicit event grammar, commit classes, promotion law
- evidence sufficiency and attestation rules
- authority, delegation, revocation, sharing, and data sovereignty
- QuerySpecification and QueryPlanIR
- PassportView / DocumentAssembly / SubmissionAssembly taxonomy
- pack/profile compatibility and merge law
- Capability Manifest as runtime self-description
- BridgeCandidate as human-gated Advisory-to-Compliance seam

Constraints:

- Do not recommend broad redesign.

Research questions:

1. What best practices exist for software-agent identity, instance identity, sponsor/accountable party, authority scope, delegation, revocation, and audit trace?
2. What fields should a governed agent run envelope contain for high-consequence workflows?
3. What trace structure explains what an agent read, assumed, called, produced, and was blocked from doing without requiring proprietary model internals?
4. Which patterns from MCP-style tool manifests, OpenAPI, AsyncAPI, Kubernetes API discovery, FHIR CapabilityStatement, FHIR OperationOutcome, OPA/Rego decision traces, W3C PROV, and similar systems are useful?
5. How should task/context handoff between agents be represented so authority does not silently move?
6. What best practices exist for world-model state, scenario state, simulation assumptions, uncertainty, invalidation, and observed-outcome reconciliation?
7. How should systems prevent simulation, prediction, confidence, recommendations, or model memory from becoming accepted operational or compliance facts?
8. What patterns are useful for AI-generated evidence/observation requests so they are actionable and not overwhelming?
9. What multi-agent/world-model scenarios are most valuable for crop farms?
10. What data-sovereignty patterns should OFARM adopt for permission-limited answers, redaction, sharing grants, revocation, benchmarking, and farm-data learning?
11. What negative tests should prove agents cannot bypass OFARM gates?
12. Which common AI-agent platform patterns should OFARM explicitly reject?
