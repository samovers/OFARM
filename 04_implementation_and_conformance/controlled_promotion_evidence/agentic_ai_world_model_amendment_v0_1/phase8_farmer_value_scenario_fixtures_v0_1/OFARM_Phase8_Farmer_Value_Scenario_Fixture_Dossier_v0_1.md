# OFARM Phase 8 — Farmer-Value Scenario Fixture Dossier v0.1

Date: 2026-05-15  
Status: SUPPORTING_REVIEW_ONLY_NOT_PROMOTED  
Phase: AAI-P8

## 1. Purpose

Phase 8 turns the agentic and world-model amendment work into farmer-facing scenario fixtures. The question for this phase is not whether an implementation exists. The question is whether the proposed contracts can support high-value crop-farm situations without weakening OFARM's active architecture.

The phase therefore creates scenario fixtures, not new law. Each fixture states:

- the farmer value goal;
- the agents, humans, and external parties involved;
- the OFARM objects and phases involved;
- the required authority, evidence, freshness, sharing, pack, and twin checks;
- the expected outputs and dispositions;
- the negative case that must be blocked or qualified.

## 2. Research integration

The attached research recommends a compact scenario set focused on evidence capture, contractor/offline reconciliation, advisory planning, storage and lot-lineage reconstruction, buyer readiness, dispute reconstruction, and controlled benchmarking. It also stresses data-sovereignty controls, event-first traceability, and local-knowledge-aware request generation.

Phase 8 maps those recommendations into OFARM scenario fixtures and explicitly connects them to Phases 3–7.

## 3. Authority classification

This phase is an implementation/conformance artifact only. It may inform future RFC promotion and implementer guidance, but it does not alter baseline law, accepted RFCs, or active machine contracts.

## 4. Quality bar

A scenario fixture is considered useful only if it contains both:

- a positive value path showing what the farmer gains; and
- a hostile path showing what OFARM must prevent.

## 5. Exit posture

Phase 8 is ready for architect review when all scenario JSON fixtures validate against the local supporting fixture schema and the validation report clearly states that runtime execution remains `NOT_RUN_NO_IMPLEMENTATION`.
