# OFARM Phase 8 — Implementer Directions v0.1

## Status

These directions are implementation guidance only. They do not create active OFARM law.

## Build guidance

An implementer should use the Phase 8 fixtures as acceptance scenarios after implementing:

- public operation descriptors;
- preflight/dry-run/explain;
- result qualification;
- runtime problem reason codes;
- trace retrieval;
- agent actorship and authority checks;
- agent run envelopes and traces;
- handoff enforcement;
- world-model advisory runtime;
- EvidenceNeed and ObservationRequest handling;
- SharingGrant and redaction enforcement.

## No-claim rule

Passing a UI mock or static fixture review is not runtime conformance. Runtime readiness requires executable tests, trace evidence, negative-case execution, and two-agent compatibility tests where applicable.
