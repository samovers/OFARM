# OFARM Farmer-Value Scenario Fixture Policy v0.1

Status: DRAFT_COMPANION_CANDIDATE_NOT_PROMOTED

## Purpose

This policy candidate defines how farmer-value scenarios should be written when testing agentic and world-model OFARM concepts.

## Rules

1. Each scenario must state the farmer benefit in operational language.
2. Each scenario must include at least one hostile case.
3. Each scenario must preserve OFARM's model/runtime split and twin separation.
4. Each scenario must mark outputs as advisory, draft, candidate, blocked, or review-required unless normal governance has accepted them.
5. Each scenario must identify which agentic phase concepts it exercises.
6. Each scenario must include no runtime readiness claim unless backed by executed conformance evidence.
