# AAI-CP9 — Farmer-value UX scenario conformance and display hardening

Date: 2026-05-17
Status: IMPLEMENTATION_CONFORMANCE_ONLY_NO_ACTIVE_PROMOTION

## Purpose

CP9 exercises farmer-facing OFARM AI workflows as conformance fixtures. It validates that useful farm workflows can keep material limitations visible without weakening active OFARM law.

CP9 does **not** add baseline law, accepted RFCs, companion artifacts, or active machine contracts. It does **not** claim farmer UX readiness, live pilot evidence, production readiness, autonomous compliance decisioning, legal advice, or external-standard readiness.

## Deep Research posture

The attached Deep Research report says farmer-facing systems should optimize for relevance, timing, trust, and low burden rather than maximal data collection. It also says farmer-facing surfaces should display qualifications before content; low-urgency EvidenceNeed/ObservationRequest traffic should be batched into daily briefs; evidence capture should support alternative routes; offline captures should be labelled `captured locally`, not `accepted`; and dispute reconstruction should be a readable timeline.

CP9 turns those findings into synthetic fixtures and a conformance runner.

## Core invariant

Farmer-facing usefulness must preserve visible qualification, authority, evidence, freshness, sharing, output disposition, request-layer weakness, and twin separation.
