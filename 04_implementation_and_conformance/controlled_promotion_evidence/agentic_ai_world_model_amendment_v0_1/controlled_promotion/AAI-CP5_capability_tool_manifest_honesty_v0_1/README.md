# AAI-CP5 — Capability/tool manifest honesty

Date: 2026-05-16
Status: CONTROLLED_PROMOTION_ACTIVE_SUBSET

## Purpose

CP5 promotes a bounded manifest-honesty layer. Manifests and tool descriptors may describe agentic capability, tool side effects, approval requirements, semantic preconditions, trace expectations, external-call posture, redaction/permission-limited result policy, data-learning posture, declared hints, and readiness-claim limits.

## Core rule

A manifest describes capability. It does not grant authority, approve a tool call, prove safety, satisfy evidence, activate a pack, publish an output, attest, file a submission, mutate either twin, or create governance success.

## Non-claims

CP5 does not claim runtime AI-agent readiness, two-agent compatibility, production readiness, autonomous compliance decisioning, world-model readiness, live registry integration, legal advice, or external-standard readiness.
