# CP4 disposition memo

Disposition: promote to active.

Rationale:

- CP2 made public surfaces, result qualification, and trace retrieval active.
- CP3 made software-agent actorship sponsor-bound and explicit.
- CP4 binds those pieces into a bounded run and handoff layer.

Quality constraints:

- handoff transfers task context, not authority;
- receiving agents must independently reauthorize and revalidate;
- tool success is separate from governance success;
- blocked actions are traceable governance outcomes;
- generated-by-agent status is provenance only;
- result qualification and trace retrieval are required for high-consequence or state-affecting runs.
