# OFARM Phase 9 Baseline Harmonization Review Checklist v0.1

Status: review aid; no authority change

## Before accepting any baseline wording

- [ ] Confirm active baseline law remains higher authority than the amendment package.
- [ ] Confirm no projection, materialization, trace, SDK, API response, or AI output becomes a truth store.
- [ ] Confirm endpoint names are not frozen as baseline law unless intentionally accepted.
- [ ] Confirm external standards are described only as implementation carriers.
- [ ] Confirm all runtime-readiness language says gates must be executed before claims.
- [ ] Confirm FMIS shadow import remains candidate-only.
- [ ] Confirm minimum capture profile remains draft/non-default.
- [ ] Confirm public/internal boundary statements do not expose platform internals.
- [ ] Confirm two-agent compatibility remains a readiness gate.
- [ ] Confirm semantic-law blockers cannot be waived by protocol conformance.

## Before promoting any draft RFC

- [ ] Check conflict with Platform Runtime and Product Architecture.
- [ ] Check conflict with Constitution truth/source-of-truth law.
- [ ] Check conflict with Authority Action Matrix.
- [ ] Check conflict with QuerySpecification and Current State RFCs.
- [ ] Check conflict with Pack Merge/Safety law.
- [ ] Check all examples retain candidate/stale/disputed/permission-limited states.

## Before promoting any machine contract

- [ ] Validate JSON Schema syntax.
- [ ] Validate bundled examples.
- [ ] Review `$id`, `schemaVersion`, required fields, and closure rules.
- [ ] Confirm schemas are classified public, internal, or conformance-only.
- [ ] Confirm public schemas do not include platform-internal mutation paths.
