# OFARM economic intelligence promotion gate and stop rules v0.1

Date: 2026-04-13
Status: active supporting implementation artifact
Scope: explicit stop rules and promotion thresholds for the bounded economics seam

---

## 1. Hard stop rules

Stop or cut back immediately if any of the following occur:

1. Scenario artifacts embed undeclared retrieval semantics that bypass QuerySpecification / QueryPlanIR.
2. Scenario results are stored or labeled as current-state authority.
3. Imported finance starts requiring ledger-like concepts to function.
4. A bridge candidate becomes effectively auto-promotable.
5. High-consequence outputs are allowed to export while stale or invalid.
6. Economics introduces a new output family instead of using existing ViewModule / ReportAssembly / DossierAssembly / SubmissionAssembly families.
7. Sharing-by-default appears for farm-private economics.
8. Utility depends on broad manual finance entry before decision support exists.

---

## 2. Promotion thresholds

### Promotion from `04` to `02/03` is blocked until all are true

- deployment-shaped proof exists, not only package-local fixtures
- projection trace-back evidence is strong enough for hostile review
- query equivalence evidence covers the actually used execution targets
- freshness/publication behavior is proven on shipped output families
- authority/sharing behavior is proven outside curated happy-path scenarios
- the seam remains generic enough to justify scenario-contract formalization rather than economics-only special pleading

### Promotion from `01 + 04` to `00` is blocked until all are true

- the concept is clearly constitutional-core rather than pack/advisory specialization
- the Alignment Register case is explicit and justified
- implementation reveals a real contradiction or missing executable constitutional contract
- hostile review says baseline change is the smallest controlled fix

---

## 3. Default decision outcomes

### Keep as-is
Use when the spike is useful and bounded but still implementation-shaped.

### Cut back
Use when the spike is valuable only as:
- views,
- bounded imported fact extracts,
- or narrower companion guidance.

### Promote narrowly
Use only when a generic scenario seam is demonstrably reusable and still anti-ERP.

### Kill
Use when economics keeps trying to recreate:
- a second truth store,
- a second query model,
- or ERP semantics.

---

## 4. Explicit non-promotion list

Do not promote any of the following during this wave:
- general ledger semantics
- AP/AR semantics
- payroll semantics
- tax/VAT semantics
- inventory valuation truth
- bank/treasury semantics
- procurement workflow semantics
- any economic current-state model
- any new passport family
